understanding this report

In this report, each function benchmarked by checkasm is assigned a section of its own. The charts in each section are active; if you hover your mouse over data points and annotations, you will see more details.

  • The chart on the left is a kernel density estimate (also known as a KDE) of time measurements. This graphs the probability of any given time measurement occurring. This function is estimated from a Log-normal distribution fitted to the measurement data. The peak corresponds to the most likely execution time (point estimate).
  • The chart on the right is the raw data from which the kernel density estimate is built. The x-axis indicates the number of loop iterations, while the y-axis shows measured execution time for the given number of loop iterations. The line behind the values is the linear regression estimate of execution time for a given number of iterations. Ideally, all measurements will be on (or very near) this line. The transparent area behind it shows the confidence interval for the execution time estimate.

Under the charts is a small table. The first two rows show the result of sampling the KDE displayed in the left-hand chart.

  • Execution cycles captures data from the underlying hardware counter.
  • Execution time is the result of normalizing the raw HW cycle count by the estimated tick frequency, and includes extra variance from the uncertainty of the estimation.
